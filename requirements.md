# 个人投资助理 - 需求文档

## 1. 项目概述

**项目名称**: Invest Assistant
**项目类型**: 个人投资研究与监控工具
**核心方法**: 基本面量化 — 将传统基本面分析与量化模型结合，筛选和监控投资标的
**目标市场**: A股、港股为主（未来扩展至美股）

---

## 2. 用户画像

| 属性 | 描述 |
|------|------|
| 技术背景 | 程序员/量化背景，熟练使用 Python，熟悉数据处理和 API 调用 |
| 投资经验 | 有基本面投资经验，理解财务指标和估值方法 |
| 隐私需求 | 高 — 数据完全本地存储，不希望上传到第三方 |

---

## 3. 功能需求

### 3.1 核心功能

#### 3.1.1 监控预警系统（优先级：高）
- **多条件监控**: 支持自定义财务指标阈值（如 PE、ROE、净利润增速等）
- **实时数据推送**: 当股票满足预设条件时，通过系统通知/邮件/Telegram 推送提醒
- **监控范围**: A股（全市场）、港股（主板、创业板）
- **监控频率**: 可配置，最小间隔 15 分钟

#### 3.1.2 选股工具（优先级：中）
- **财务筛选器**: 按多个财务指标组合筛选股票
  - 盈利能力: ROE、毛利率、净利率
  - 成长性: 营收增速、净利润增速
  - 估值: PE、PB、PS
  - 财务健康: 资产负债率、经营现金流
- **自定义筛选条件**: 用户可保存和复用筛选模板

#### 3.1.3 估值分析（优先级：中）
- **估值模型**: DCF、DDM、PE/PB 相对估值
- **估值区间计算**: 基于敏感分析输出乐观/中性/悲观估值
- **估值报告**: 生成可视化估值图表

#### 3.1.4 回测系统（优先级：中）
- **事件驱动回测**: 支持基于财务数据的策略回测
- **回测指标**: 年化收益、最大回撤、夏普比率、胜率等
- **回测报告**: 生成可视化图表和详细统计
- **数据要求**: 需要完整的历史财务数据

#### 3.1.5 模拟交易（优先级：低）
- **模拟撮合**: 基于每日收盘价模拟成交
- **持仓管理**: 记录模拟持仓、计算浮动盈亏
- **交易日志**: 记录每笔模拟交易的时间、价格、数量
- **收益统计**: 计算模拟组合的收益率和风险指标

#### 3.1.6 组合管理（优先级：低）
- 持仓记录与跟踪
- 收益计算与统计
- 交易记录管理

### 3.2 数据来源

| 数据类型 | 来源 | 说明 |
|----------|------|------|
| 主要数据源 | TickFlow | 市场覆盖: A股/港股/美股/期货，免费层支持日线+财务数据 |
| 日线行情 | TickFlow | 日K、周K、月K |
| 财务数据 | TickFlow | 财务报表(资产负债表/利润表/现金流量表)、财务指标 |
| 历史数据 | 本地存储 | 定期同步到本地数据库 |

**数据存储**: 完全本地化，使用 PostgreSQL 存储

**TickFlow 免费层覆盖**:
- A股: 全市场（上交所/深交所/北交所）
- 港股: 主板、创业板
- 美股: 主要交易所
- 日线数据: 历史数据
- 财务数据: 季报/年报财务指标、财报原始数据

**付费可扩展**: 实时行情、分钟线数据（按需启用）

## 4. 技术架构

### 4.1 技术栈

```
前端:       Web UI (可选) / API
后端:       Python (FastAPI)
数据库:     PostgreSQL + Redis
数据获取:   JoinQuant（聚宽）API
任务调度:   APScheduler
通知:       系统通知 / 邮件 / Telegram Bot
```

### 4.2 核心模块

```
invest_assistant/
├── api/              # API 接口
├── core/             # 核心逻辑
│   ├── monitor/      # 监控引擎
│   ├── screener/     # 选股器
│   ├── valuation/    # 估值模型
│   ├── backtest/    # 回测引擎
│   └── simulation/   # 模拟交易
├── data/             # 数据获取与存储
│   ├── loader/      # 数据加载（TickFlow API）
│   └── storage/     # 本地数据库（PostgreSQL）
├── notification/     # 通知推送
├── config/           # 配置管理
└── models/           # 数据模型

数据库详情: 参见 db_schema.md
```
