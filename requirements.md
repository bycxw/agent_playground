# 个人投资助理 - 需求文档

## 1. 项目概述

**项目名称**: Invest Assistant
**项目类型**: 个人投资研究与监控工具
**核心方法**: 基本面量化 — 将传统基本面分析与量化模型结合，筛选和监控投资标的
**目标市场**: A股、港股为主（未来扩展至美股）
**开发基础**: 基于开源框架 [zvt](https://github.com/zvtvz/zvt) 构建

---

## 2. 用户画像

| 属性 | 描述 |
|------|------|
| 技术背景 | 程序员/量化背景，熟练使用 Python，熟悉数据处理和 API 调用 |
| 投资经验 | 有基本面投资经验，理解财务指标和估值方法 |
| 隐私需求 | 高 — 数据完全本地存储，不希望上传到第三方 |

## 3. 功能需求

### 3.1 核心功能

#### 3.1.1 监控预警系统（优先级：高）
- **多条件监控**: 支持自定义财务指标阈值（如 PE、ROE、净利润增速等）
- **实时数据推送**: 当股票满足预设条件时，通过系统通知/邮件/Telegram 推送提醒
- **监控范围**: A股（全市场）、港股（主板、创业板）
- **监控频率**: 可配置，最小间隔 15 分钟

#### 3.1.2 选股工具（优先级：中）
- **财务筛选器**: 按多个财务指标组合筛选股票
  - 盈利能力: ROE、毛利率、净利率
  - 成长性: 营收增速、净利润增速
  - 估值: PE、PB、PS
  - 财务健康: 资产负债率、经营现金流
- **自定义筛选条件**: 用户可保存和复用筛选模板

#### 3.1.3 估值分析（优先级：中）
- **估值模型**: DCF、DDM、PE/PB 相对估值
- **估值区间计算**: 基于敏感分析输出乐观/中性/悲观估值
- **估值报告**: 生成可视化估值图表

#### 3.1.4 回测系统（优先级：中）
- **事件驱动回测**: 支持基于财务数据的策略回测
- **回测指标**: 年化收益、最大回撤、夏普比率、胜率等
- **回测报告**: 生成可视化图表和详细统计
- **数据要求**: 需要完整的历史财务数据

#### 3.1.5 模拟交易（优先级：低）
- **模拟撮合**: 基于每日收盘价模拟成交
- **持仓管理**: 记录模拟持仓、计算浮动盈亏
- **交易日志**: 记录每笔模拟交易的时间、价格、数量
- **收益统计**: 计算模拟组合的收益率和风险指标

#### 3.1.6 组合管理（优先级：低）
- 持仓记录与跟踪
- 收益计算与统计
- 交易记录管理

## 4. 技术架构

### 4.1 技术栈

| 组件 | 技术 | 说明 |
|------|------|------|
| **数据框架** | zvt | 开源量化框架，负责数据获取、存储、查询 |
| **数据源** | eastmoney（东方财富） | 免费，无需 API Key |
| **API 框架** | FastAPI | 提供 REST API 接口 |
| **任务调度** | APScheduler | 定时执行监控和数据同步 |
| **数据库** | SQLite + zvt 内置存储 | 业务数据用 SQLite，行情数据由 zvt 管理 |
| **通知** | Email / Telegram Bot | 告警推送 |

### 4.2 项目结构

```
invest_assistant/
├── src/invest_assistant/
│   ├── main.py              # FastAPI 入口 + 定时任务
│   ├── api/                 # API 接口
│   │   ├── monitor.py       # 监控 API
│   │   ├── stocks.py        # 股票查询 API
│   │   └── sync.py          # 数据同步 API
│   ├── core/                # 核心业务逻辑
│   │   ├── monitor/         # 监控引擎
│   │   └── notification/    # 通知推送
│   ├── data/                # 数据层（封装 zvt）
│   │   ├── provider.py      # 数据查询封装
│   │   └── sync.py          # 数据同步任务
│   ├── services/            # 业务服务（调用 zvt）
│   │   └── screener.py      # 选股服务
│   ├── models/              # 数据模型
│   └── config.py            # 配置管理
├── data/                    # 数据存储目录
│   ├── zvt/                 # zvt 数据（eastmoney 数据源）
│   └── my/                  # 业务数据
└── tests/
```

### 4.3 与 zvt 的关系

| 功能 | 实现方式 |
|------|----------|
| 数据获取 | zvt.recorders（东方财富数据源） |
| 数据查询 | zvt.api（query_kdata、query_finance） |
| 选股 | zvt.factors.TargetSelector + 自定义筛选 |
| 回测 | zvt.trader |
| 监控引擎 | **自己开发** |
| 通知推送 | **自己开发** |
| API 接口 | **自己开发**（FastAPI） |

## 8. 数据存储说明

### zvt 数据存储

zvt 自动管理行情和财务数据的存储，位置：`data/zvt/`

| 数据类型 | 存储位置 | 说明 |
|----------|----------|------|
| 股票列表 | zvt 自动管理 | 通过 EastmoneyStockMetaRecorder 获取 |
| 日线行情 | zvt 自动管理 | 通过 EastmoneyKdataRecorder 获取 |
| 财务指标 | zvt 自动管理 | 通过 EastmoneyFinanceFactorRecorder 获取 |

### 业务数据存储

自己管理的业务数据，位置：`data/my/invest_assistant.db`

| 表 | 说明 |
|----|------|
| 无需建表 | 监控规则和事件目前存于内存，暂未持久化 |
